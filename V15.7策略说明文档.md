## 二、策略核心逻辑

### 2.1 一句话概括

每3个交易日，从10只ETF中选动量最强的N只持有，动量为负时自动空仓或持有国债，ATR跟踪止损保底。

### 2.2 交易流程（每个交易日09:35执行）

```
第一步：止损检查
  → 遍历所有持仓，价格跌破ATR止损线的立即卖出

第二步：轮动日判断
  → 每3个交易日触发一次完整轮动（或止损触发时提前轮动）

第三步：计算动量排名
  → 对10只ETF分别计算动量得分，过滤不合格的（动量为负/价格在MA20下方）

第四步：确定目标持仓
  → 取排名前N只作为目标
  → 如果候选不足N只，用国债ETF填空

第五步：卖出
  → 持仓中不在目标列表的，轮动卖出

第六步：买入
  → 目标列表中未持有的，按仓位规则买入
```

### 2.3 动量得分计算

对每只ETF，基于T-1日收盘数据计算：

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| ROC20（短期动量） | `close[-1] / close[-20] - 1` | 过滤+排名 |
| ROC60（中期动量） | `close[-1] / close[-60] - 1` | 过滤 |
| 波动率 | `20日收益率标准差 × √252` | 风险调整+仓位 |
| 风险调整动量 | `ROC20 / 波动率` | 排名核心 |
| MA20 | `20日收盘均价` | 趋势确认 |
| 趋势强度 | `(收盘价 - MA20) / MA20` | 排名辅助 |

**过滤条件（全部满足才入选）：**
- ROC20 > 0（短期动量为正）
- ROC60 > 0（中期动量为正）
- 收盘价 > MA20（价格在均线上方）

**排名公式：**
```
sort_score = 风险调整动量 × 0.7 + 趋势强度 × 0.3
```

### 2.4 ATR跟踪止损

| 参数 | 值 | 说明 |
|------|---|------|
| ATR周期 | 14日 | Wilder经典参数 |
| 正常倍数 | 2.5× | 年化波动率≤30%时 |
| 高波动倍数 | 2.0× | 年化波动率>30%时，收紧止损 |
| 止损下限 | 3% | 防噪音震荡 |
| 止损上限 | 15% | 防单笔巨亏 |

**止损价 = 买入后最高价 × (1 - 止损百分比)**

最高价使用收盘价更新（非盘中最高价），每日15:00更新一次。

### 2.5 国债兜底与填空

| 场景 | 行为 |
|------|------|
| 0只ETF通过动量过滤 | 卖出所有持仓，全仓买入国债ETF（511010） |
| 1-2只通过过滤（max_hold=3时） | 买入通过的ETF + 国债ETF填满剩余仓位 |
| 3只以上通过过滤 | 买入排名前3，不涉及国债 |
| 国债自身通过动量过滤 | 正常参与排名，不重复填空 |

注：回测验证国债填空在11年中从未触发——国债ETF因低波动正动量的特性，在权益类ETF大面积弱势时自然进入排名前N。该机制作为逻辑安全网保留。

### 2.6 仓位管理

**资金档位：**

| 档位 | 总资产 | 最大持仓 | 基础仓位比例 |
|------|--------|---------|-------------|
| micro | <1.5万 | 1 | 90% |
| small | 1.5-5万 | 2 | 80% |
| medium | 5-10万 | 3 | 70% |
| large | >10万 | 3 | 60% |

**单只ETF仓位计算：**
```
基础分配 = 可用资金 / 空余仓位数 × base_position_ratio

波动率调整 = target_vol(15%) / actual_vol
             限制在 [0.4, 1.5] 范围内

最终分配 = 基础分配 × 波动率调整
           不超过可用资金的95%
           国债填空标的不做波动率调整
```

## 六、聚宽版 vs PTrade版 差异对照

### 6.1 平台适配（必要差异，不影响逻辑）

| 项目 | 聚宽 | PTrade |
|------|------|--------|
| 代码格式 | `.XSHG` / `.XSHE` | `.SS` / `.SZ` |
| 总资产 | `context.portfolio.total_value` | `context.portfolio.portfolio_value` |
| 可用资金 | `context.portfolio.available_cash` | `context.portfolio.cash` |
| 持仓数量 | `pos.total_amount` | `pos.amount` |
| 持仓成本 | `pos.avg_cost` | `pos.cost_basis` |
| 持仓现价 | `pos.price` | `pos.last_sale_price` |
| 当前价 | `get_current_data()[code].last_price` | `get_snapshot(code)['last_px']` / `data[code].price` |
| 停牌检测 | `current_data[code].paused` | `get_stock_status([code], 'HALT')` |
| 调度函数 | `run_daily(func, time)` | `run_daily(context, func, time)` |
| 导入 | `from jqdata import *` | `from datetime import datetime, timedelta` |

### 6.2 PTrade特有的安全机制

| 机制 | 说明 | 聚宽是否需要 |
|------|------|-------------|
| `sold_today` 防重复卖出 | PTrade持仓同步有6秒延迟，同一标的可能被重复卖出 | 不需要 |
| `_validate_price` 价格校验 | 实盘中get_snapshot可能返回异常价格，偏差>15%跳过 | 不需要 |
| `skip_paused` 回退处理 | PTrade可能不支持该参数，回退后手动过滤成交量=0的停牌日 | 不需要（聚宽原生支持） |
| 三级价格获取 | `get_snapshot` → `data[code].price` → `get_history` 逐级回退 | 不需要（聚宽API稳定） |

### 6.3 回测行为差异（PTrade框架限制）

| 行为 | 聚宽回测 | PTrade回测 | PTrade实盘 |
|------|---------|-----------|-----------|
| 止损检查时间 | 09:35（盘中价） | 15:00（收盘价） | 09:35（盘中价） |
| 买入成交价 | 09:35实时价 | 收盘价 | 09:35实时价 |
| 最高价更新 | 15:00 | 15:00（同一pass） | 15:00 |
| run_daily时间 | 按设定执行 | **全部固定15:00** | 按设定执行 |

**结论：PTrade回测结果不反映策略真实收益，仅用于验证代码无报错。策略收益以聚宽回测为准。PTrade实盘行为与聚宽一致。**

### 6.4 策略逻辑一致性确认

| 逻辑模块 | 是否一致 | 备注 |
|----------|---------|------|
| ETF池（10只） | 一致 | 仅代码格式不同 |
| 资金档位（4级） | 一致 | 参数完全相同 |
| 动量计算（ROC20/60/MA20/vol） | 一致 | |
| 排名公式（risk_adj×0.7 + trend×0.3） | 一致 | |
| 双重动量过滤（ROC20>0, ROC60>0, 价格>MA20） | 一致 | |
| ATR跟踪止损（动态2.0x/2.5x） | 一致 | PTrade增加价格校验 |
| 止损参数（floor=3%, cap=15%） | 一致 | |
| 国债兜底 + 国债填空 | 一致 | |
| 买入价使用实时价 | 一致 | 聚宽用last_price，PTrade用get_snapshot |
| 波动率反比仓位调整 | 一致 | |
| 国债填空跳过波动率调整 | 一致 | |

## 九、部署指南

### 9.1 推荐工作流

```
步骤1：聚宽回测 → 验证策略收益（已完成，212.8%）
步骤2：PTrade回测 → 验证代码无报错（不看收益数字）
步骤3：PTrade模拟 → 2-4周，确认信号与聚宽一致
步骤4：实盘小资金 → 1-2万，跑2-3个月
步骤5：实盘加仓 → 确认无问题后加到目标金额
```

### 9.2 PTrade实盘注意事项

| 事项 | 说明 |
|------|------|
| run_daily总数 | 策略用了4个（09:30/09:35/15:00/15:30），剩余1个余量 |
| order_target延迟 | 持仓同步有6秒延迟，已用sold_today防护 |
| QDII额度 | 纳指/标普500/恒生ETF可能遇到申购限制，溢价时策略仍会买入 |
| g变量持久化 | g自动pickle，`__`前缀变量（如`g.__is_live`）不持久化 |
| 重启恢复 | initialize先执行，然后持久化数据覆盖。day_count等状态会保留 |

### 9.3 实盘监控要点

| 监控项 | 方法 | 频率 |
|--------|------|------|
| 信号一致性 | 对比PTrade日志与聚宽同期信号 | 每周 |
| 收益偏差 | PTrade实盘 vs 聚宽同期回测，偏差>5%排查 | 每月 |
| 止损触发 | 检查止损价格是否合理，有无异常价格触发 | 每次止损后 |
| 国债角色 | 确认弱势时国债正常买入 | 每月 |
