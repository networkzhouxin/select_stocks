## 二、策略核心逻辑

### 2.1 一句话概括

每天从12只ETF中用ROC+LR混合评分选最强的3只持有，双重过滤抑制噪音，ATR跟踪止损保底。

### 2.2 交易流程（每个交易日09:35执行）

```
第一步：止损检查
  → 遍历所有持仓，价格跌破ATR止损线的立即卖出

第二步：每日轮动（rebalance_interval=1）
  → 每天都执行完整轮动逻辑（消除路径偏移）

第三步：计算动量排名（ROC+LR混合评分）
  → 对12只ETF分别计算ROC评分和LR评分
  → 双重过滤：ROC过滤 + LR×R²过滤（交集）
  → 分位数归一化后各50%权重合成排名

第四步：确定目标持仓
  → 取排名前3只作为目标
  → 如果候选不足3只，用国债ETF填空

第五步：卖出
  → 持仓中不在目标列表的，轮动卖出

第六步：买入
  → 目标列表中未持有的，按仓位规则买入
```

### 2.3 混合动量评分

对每只ETF，基于T-1日收盘数据计算两套评分：

**ROC评分维度：**

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| ROC20（短期动量） | `close[-1] / close[-20] - 1` | 过滤+评分 |
| ROC60（中期动量） | `close[-1] / close[-60] - 1` | 过滤 |
| 波动率 | `20日收益率标准差 × √252` | 风险调整+仓位 |
| 风险调整动量 | `ROC20 / 波动率` | ROC评分核心 |
| MA20 | `20日收盘均价` | 趋势确认 |
| 趋势强度 | `(收盘价 - MA20) / MA20` | ROC评分辅助 |

**LR评分维度（线性回归）：**

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| 对数价格线性回归 | `np.polyfit(x, log(prices), 1, w=weights)` | 趋势方向和速度 |
| 年化收益率 | `exp(slope × 252) - 1` | 趋势强度 |
| R²（拟合优度） | `1 - ss_res/ss_tot`（加权） | 趋势纯度 |
| LR评分 | `年化收益率 × R²` | 趋势强度×纯度 |

**过滤条件（交集，全部满足才入选）：**
- ROC20 > 0（短期动量为正）
- ROC60 > 0（中期动量为正）
- 收盘价 > MA20（价格在均线上方）
- LR×R² > 0（趋势纯度为正）

**排名公式（分位数归一化后等权）：**
```
roc_score = 风险调整动量 × 0.7 + 趋势强度 × 0.3
lr_score = 年化收益率 × R²

对所有通过过滤的ETF：
  _roc_rank = roc_score的分位数排名（0到1）
  _lr_rank = lr_score的分位数排名（0到1）
  最终评分 = _roc_rank × 0.5 + _lr_rank × 0.5
```

分位数归一化确保两个评分维度等权贡献排名，不受量纲差异影响。

### 2.4 ATR跟踪止损

| 参数 | 值 | 说明 |
|------|---|------|
| ATR周期 | 14日 | Wilder经典参数 |
| 正常倍数 | 2.5× | 年化波动率≤30%时 |
| 高波动倍数 | 2.0× | 年化波动率>30%时，收紧止损 |
| 止损下限 | 3% | 防噪音震荡 |
| 止损上限 | 15% | 防单笔巨亏 |

**止损价 = 买入后最高价 × (1 - 止损百分比)**

最高价使用收盘价更新（非盘中最高价），每日15:30更新一次（PTrade实盘合并到after_close中，确保收盘集合竞价数据落定）。

### 2.5 国债填空

| 场景 | 行为 |
|------|------|
| 0只ETF通过动量过滤 | 国债填空，用95%可用资金买入国债ETF |
| 1-2只通过过滤 | 买入通过的ETF + 国债ETF填空剩余仓位 |
| 3只以上通过过滤 | 买入排名前3，不涉及国债 |
| 国债自身通过动量过滤 | 正常参与排名，不重复填空 |

注：回测验证国债填空在11年中从未触发——国债ETF因低波动正动量的特性，在权益类ETF大面积弱势时自然进入排名前N。该机制作为逻辑安全网保留。

### 2.6 仓位管理

**资金档位（V15.9：统一max_hold=3）：**

| 档位 | 总资产 | 最大持仓 | 基础仓位比例 |
|------|--------|---------|-------------|
| micro | <1.5万 | 3 | 70% |
| small | 1.5-5万 | 3 | 70% |
| medium | 5-10万 | 3 | 65% |
| large | >10万 | 3 | 60% |

**单只ETF仓位计算：**
```
权益类标的：
  基础分配 = 可用资金 / 空余仓位数 × base_position_ratio
  波动率调整 = target_vol(15%) / actual_vol，限制在 [0.4, 1.5]
  最终分配 = min(基础分配 × 波动率调整, 可用资金 × 95%)

国债填空标的：
  最终分配 = 可用资金 × 95%（尽量多买，不做波动率调整）
```

### 6.2 PTrade特有的安全机制

| 机制 | 说明 |
|------|------|
| `sold_today` 防重复卖出 | PTrade持仓同步有6秒延迟，同一标的可能被重复卖出 |
| `skip_paused` 回退处理 | PTrade不支持该参数，手动过滤volume=0的停牌日 |
| 三级价格获取 | `get_snapshot` → `data[code].price` → `get_history` 逐级回退 |
| limit_price传参 | 所有order/order_target传limit_price，避免PTrade内部snapshot失败 |
| 跌停价卖出 | 止损/轮动卖出用down_px作limit_price，确保下跌行情成交 |
| QDII溢价过滤 | iopv溢价>5%时跳过买入，防止高溢价接盘 |
| 涨停过滤 | check_limit检测涨停，避免挂单浪费资金 |
| on_trade_response | 成交回调确认后才设止损基准，避免未成交盲设 |
| on_order_response | 委托失败检测，卖出失败重置sold_today允许重试 |
| ATR恢复机制 | PTrade重启后自动从历史数据重建止损基准 |
| snapshot缓存 | 一次get_snapshot复用trade_status/iopv/down_px |
| set_universe | 注册标的池，确保data[code]可访问 |

### 9.2 PTrade模拟交易说明

#### 模拟启动时的正常提示

首次启动模拟/实盘时会出现以下WARNING，属于正常现象：

```
WARNING - 交易不支持set_commission函数
WARNING - 交易不支持set_slippage函数
```

原因：`set_commission`和`set_slippage`仅回测生效，实盘/模拟的佣金由券商设置、滑点由真实市场决定。代码已用`try/except`包裹，不影响策略运行。

#### 模拟资金与实盘资金不同

PTrade模拟的初始资金通常较大且不可调。这**不影响信号验证**：

| 模块 | 是否受资金影响 |
|------|-------------|
| 动量排名（选哪3只ETF） | 不受影响 — 排名只看价格动量 |
| 止损触发（什么时候卖） | 不受影响 — 只看价格回撤百分比 |
| 轮动时机（什么时候换仓） | 不受影响 — 每日轮动 |
| 档位（base_ratio） | 受影响 — 大资金在large档(0.60)，小资金在small档(0.70) |
| 每只ETF买多少股 | 受影响 — 大资金买更多股，但买哪只不变 |

#### 模拟期间验证要点

模拟不是为了验证收益数字，而是验证：

1. **信号是否正确** — 对比PTrade模拟日志和聚宽同日信号，应该买卖同样的ETF
2. **代码是否正常运行** — 09:30/09:35/15:30三个时间点是否按时触发
3. **下单是否成功** — 有没有报错、挂单失败、数据获取异常

以上验证项与资金大小无关。模拟2-4周确认无问题，即可用自己的资金上实盘。

### 9.3 PTrade实盘注意事项

| 事项 | 说明 |
|------|------|
| run_daily总数 | 策略用了3个（09:30/09:35/15:30），剩余2个余量 |
| order_target延迟 | 持仓同步有6秒延迟，已用sold_today防护 |
| QDII额度 | 纳指/标普/恒生/日经/中概互联可能遇到申购限制或溢价，PTrade版已加iopv溢价>5%自动跳过 |
| g变量持久化 | g自动pickle，`__`前缀变量（如`g.__is_live`）不持久化 |
| 重启恢复 | initialize先执行，然后持久化数据覆盖。day_count等状态会保留 |

### 9.4 实盘监控要点

| 监控项 | 方法 | 频率 |
|--------|------|------|
| 信号一致性 | 对比PTrade日志与聚宽同期信号 | 每周 |
| 收益偏差 | PTrade实盘 vs 聚宽同期回测，偏差>5%排查 | 每月 |
| 止损触发 | 检查止损价格是否合理 | 每次止损后 |
| QDII溢价 | PTrade版已自动跳过>5%溢价，但仍需关注长期溢价趋势 | 每周 |
| 15:30日志现价 | PTrade的last_sale_price可能与券商APP收盘价差几分钱，属正常 | 了解即可 |

### 9.5 中途停止与资金变动

#### 提取资金

策略会自动适应。`update_tier`每天09:30读取总资产，提钱后总资产下降，自动触发档位变化。但V15.9统一了max_hold=3，档位变化只影响base_ratio（0.70/0.65/0.60），影响很小。买入时用可用资金计算，提钱后自然少买。已持有的仓位不受影响，继续按止损和轮动规则管理。

#### 追加资金

与提取资金同理，总资产增加后档位可能升级（如small→medium），base_ratio微调。策略会在下次轮动日自动用新增资金参与买入。

#### 暂停策略再重启

PTrade会自动pickle `g`变量（`__`前缀变量除外）。重启后关键状态会恢复：

| 变量 | 是否恢复 | 说明 |
|------|---------|------|
| `g.day_count` | 恢复 | 轮动节奏继续 |
| `g.highest_since_buy` | 恢复 | 止损基准还在 |
| `g.entry_atr` | 恢复 | 止损计算正常 |
| `g.current_tier` | 恢复 | 档位状态 |
| `g.__is_live` | **不恢复** | `__`前缀排除持久化，initialize中重新检测 |

**关闭期间不做任何操作 → 重启后没问题。** 持仓还在，`g`变量恢复，15:00的`update_highest`会在第一天把最高价更新到最新，次日起恢复正常交易。

**关闭期间手动买卖了 → 有隐患：**

| 手动操作 | 风险 | 严重程度 |
|---------|------|---------|
| 手动卖出某只ETF | `g.highest_since_buy`残留旧数据，但代码只遍历实际持仓，不会出错 | 无影响 |
| 手动买入新ETF | 新仓位没有止损记录，直到15:00 `update_highest`补上 | 低风险，最多1天无止损 |
| 手动提现大量资金 | 持仓不变但可用资金骤降，下次轮动买不起新标的 | 低风险，不影响现有持仓 |

#### 推荐操作方式

```
暂停策略：
  → 在PTrade中停止策略即可，不要手动平仓
  → 持仓继续由券商管理，只是没有自动交易

恢复策略：
  → 在PTrade中重新启动（或新增策略从回测中选择）
  → before_trading_start自动执行_recover_missing_atr恢复止损基准
  → 15:30 update_highest 补全最高价记录
  → 次日09:35起恢复正常交易

更换策略版本：
  → 收盘后（15:30之后）停止旧策略
  → 在回测模块粘贴新代码并回测
  → 在交易模块新增策略，从回测中选择
  → 启动新策略，旧策略持仓会被自动接管

提钱/加钱：
  → 直接操作即可，不需要停止策略
  → 策略会自动适应剩余/新增资金量
  → 注意：提钱过多导致仓位占比过高时，策略不会主动减仓，只是不再新买
```

**核心原则：要停就停策略本身，不要边停边手动操作。**

### 9.6 单独回测注意事项

不要单独回测某一年（如只测2020年）。动量策略需要前序持仓状态，单独回测从空仓开始会导致：
- 前几天在"找方向"，可能错过行情
- day_count从0重置，轮动节奏不对
- 资产规模从初始值开始，档位不同

正确做法：始终从2015年开始连续回测，看某一年在连续回测中的表现。

### 核心教训总结

1. **跨资产分散是收益翻倍的关键** — 4+3+3 → 4+5+3 持续有效
2. **行业ETF不如宽基** — 高波动≠高收益，只是更多止损摩擦
3. **A股不可替代** — 动量框架自动在A股弱势时减仓，人为去掉只会丧失牛市机会
4. **止损后的"鞭打"买回是正确行为** — 冷却期阻止再入场会损失41pp
5. **小资金也应持3只** — ETF价格足够低，统一max_hold=3消除结构性劣势
6. **LOF基金不适合动量轮动** — 场内流动性差导致订单失败
7. **简单优于复杂** — 过了复杂度天花板后，每多一个规则大概率减分
8. **信号驱动和动量轮动不能共享ETF池** — 两种框架的最优池子完全不同
9. **再平衡时机运气是真实风险** — 同一策略不同起点差距可达161pp，选择确定性方案比追求最高收益更理性
10. **ROC+LR交集过滤有效抑制噪音** — 在每日轮动场景下，双重评分比单一评分减少20%无效换仓

## 十二、实盘预期与风险提示

### 12.1 合理预期

| 指标 | 回测值 | 实盘预期 | 差距原因 |
|------|--------|---------|---------|
| 年化收益 | ~8.3% | 6-8% | QDII溢价、实际滑点、流动性 |
| 最大回撤 | — | 15-20% | 未来黑天鹅无法回测 |
| 单年最大亏损 | — | -5%到-10% | 实盘心态+执行偏差 |
| 连续亏损年数 | — | 可能2-3年 | 市场环境变化 |
| 路径偏移 | **0（确定性）** | **0** | Hybrid每日轮动消除 |

### 12.2 主要风险

| 风险 | 说明 | 应对 |
|------|------|------|
| QDII溢价/限购 | 纳指/标普/日经/中概互联等QDII ETF可能出现大幅溢价或申购暂停 | PTrade版已加iopv溢价>5%自动跳过 |
| 策略拥挤 | 动量策略被越来越多人使用，alpha可能衰减 | ETF市场容量大，短期影响小 |
| 市场结构变化 | 未来市场规律可能与过去11年不同 | 动量因子30年有效，但不保证永远有效 |
| 心态风险 | 连续止损时能否严格执行，这是最大的敌人 | 用PTrade自动交易，排除人为干预 |
| 极端事件 | 2015级别的股灾、全球金融危机 | ATR止损+国债兜底是安全网，但无法完全消除尾部风险 |

### 12.3 核心纪律

1. **不要手动干预** — 看到亏损不要手动平仓，看到大涨不要手动加仓
2. **至少给策略1年时间** — 2021和2022连续微亏，扛住才能等到2025年的+25%
3. **不要根据短期表现判断策略好坏** — 任何3个月内的表现都可能是随机噪音
4. **定期复盘但不频繁调参** — 每月对比一次实盘vs聚宽，偏差>5%才排查
5. **资金管理优先** — 不要投入超过总资产20%的钱到单一策略

### 12.4 什么时候应该停止策略

| 信号 | 说明 |
|------|------|
| 连续12个月收益显著低于聚宽同期 | 可能有执行层面的问题 |
| QDII ETF长期溢价>10% | 实盘买入成本系统性偏高 |
| 止损率突然飙升到40%+ | 可能市场结构发生变化 |
| 策略连续3年亏损 | 需要重新评估框架有效性 |

**以上信号出现时不要立即停止，先排查原因。大多数情况下是短期波动，不是策略失效。**

### 12.5 回测佣金与实盘佣金的差异

| | 回测设置 | 实盘实际 |
|--|---------|---------|
| 费率 | 万分之3 | 万分之0.5 |
| 最低收费 | 5元 | 0.5元 |
| 差距 | — | **实盘成本约为回测的1/10** |

聚宽强制min_commission≥5元，无法调低。这意味着**回测收益系统性低估了实盘收益**——实盘每笔交易少扣约4.5元，1000+次交易累计可多出约4,500元（占2万本金22.5%）。实盘实际年化预期比回测高2-4个百分点。

### 12.6 近期表现与长期预期

| 区间 | 总收益 | 年化 | 说明 |
|------|--------|------|------|
| 2015-2026（11年） | 142.5% | 8.3% | 完整回测基准 |
| 2020-2025（6年） | 30.2% | 4.5% | 近期表现偏弱 |
| 2021-2025（5年） | 2.3% | 0.5% | 回测佣金拖累，实盘预期+24pp |

近5年回测收益偏低的原因：
1. **回测佣金是实盘的10倍**（5元 vs 0.5元），小资金影响巨大
2. **2021-2022全球放水单边牛市**，动量轮动不如买入持有
3. **2023-2024市场波动率低**，资产分化度不足，轮动优势不明显

但当前环境（2026年霍尔木兹海峡危机、全球波动率飙升、资产分化加剧）恰好是动量轮动的有利环境。

### 12.7 策略不适用的场景

| 场景 | 不适用 |
|------|--------|
| 个股交易 | 个股可退市/停牌/暴雷，策略参数针对ETF校准 |
| 追求暴利（年化50%+） | 策略年化预期6-8%，靠本金和复利增长 |
| 无法忍受连续亏损 | 可能连续2-3年微亏，必须扛住 |
| 频繁手动干预 | 任何手动操作都会破坏策略的统计优势 |
